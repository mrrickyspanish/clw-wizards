# SEO Standards

Version: 2.0

## Purpose

These standards define how AI agents should handle SEO work for RJ's projects.

SEO must support human clarity and business value. It should not make copy robotic.

## SEO philosophy

Good SEO helps the right person understand that the page matches their need.

It should improve:

- Page purpose
- Search intent alignment
- Headings
- Metadata
- Internal links
- Alt text
- Local relevance when appropriate
- Service specificity

It should not create keyword stuffing or bloated pages.

## Page title standard

A good page title should:

- Be unique
- Describe the page clearly
- Include the primary service or topic naturally
- Include brand or business name when appropriate
- Avoid vague slogans

Bad title:

- Home
- Services
- Creative Solutions for Every Brand
- Elevate Your Digital Presence

Better title patterns:

- Website Design for Small Businesses | Brand Name
- Social Media Content Services | Brand Name
- Cruise Travel Tips and Planning Resources | Brand Name
- Cabinet Refinishing in Charlotte | Brand Name

## Meta description standard

A good meta description should:

- Summarize the page clearly
- Include the service or topic naturally
- Speak to the audience
- Suggest a useful next step
- Avoid stuffing

Good meta descriptions are written for humans first.

## H1 standard

Each page should have one clear H1.

The H1 should state the page's purpose.

Avoid vague H1s that only sound clever.

## H2 standard

H2s should help users and search engines understand the page structure.

Good H2s should:

- Label meaningful sections
- Use natural language
- Include service or topic language where appropriate
- Avoid empty phrases like "What We Do" when a more specific heading would help

## Search intent standard

Before editing a page, identify likely intent:

- Informational
- Commercial
- Local service
- Brand trust
- Contact or conversion
- Product shopping
- Portfolio proof

Copy should match the visitor's intent.

## Internal linking standard

Use internal links to:

- Connect related services
- Move users toward contact
- Connect work examples to services
- Connect educational pages to inquiry paths
- Help search engines understand site structure

Avoid:

- Random internal links
- Overlinking
- Vague "click here" links
- Links that distract from conversion

## Alt text standard

Alt text should describe meaningful images.

Good alt text:

- Describes the image accurately
- Includes context when useful
- Avoids keyword stuffing
- Does not describe decorative images unless needed

## Local SEO standard

Use local language only where natural and relevant.

Do not force location terms into every page.

Good local use:

- Service area page
- Local service client
- About page with actual location context
- Contact page
- Local landing page

Bad local use:

- Repeating city names unnaturally
- Stuffing suburbs into every paragraph
- Adding location terms where the page is not location-focused

## Service keyword standard

Use service language naturally.

Examples:

- Website design for small businesses
- Small business website design
- Website strategy and design
- Digital marketing for small businesses
- Social media content for businesses
- Business website design and development
- Digital presence for service businesses

Do not force exact phrases if they make copy worse.

## SEO audit output

Use this format:

### Search Intent

State the likely intent.

### Current Issues

List SEO weaknesses.

### Recommended Changes

Prioritize changes.

### Implemented Changes

List what was changed.

### Metadata

Show page title and meta description if relevant.

### Internal Links

List links added or recommended.

### Risks

Call out assumptions or anything not verified.
