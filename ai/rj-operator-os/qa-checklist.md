# QA Checklist

Version: 2.0

Copy this file into the working session and check items off. Every unchecked item must be explained in the final report.

Use this checklist before reporting that meaningful work is complete.

## Scope check

- [ ] Did the work stay within the mission?
- [ ] Were unrelated files avoided?
- [ ] Were unrelated pages avoided?
- [ ] Were no major new sections added unless requested?
- [ ] Were no offers, pricing, or business rules changed unless requested?
- [ ] Were no destructive actions taken?

## Build and code check

- [ ] Build was run, or reason not run is stated.
- [ ] Lint was run, or reason not run is stated.
- [ ] Type check was run, or reason not run is stated.
- [ ] Tests were run if relevant, or reason not run is stated.
- [ ] No new errors were introduced knowingly.
- [ ] No unnecessary dependencies were added.
- [ ] No environment variables or secrets were touched unless explicitly requested.

## Git check

- [ ] Correct branch used.
- [ ] Branch name reported.
- [ ] No merge to main unless instructed.
- [ ] No production promotion unless instructed.
- [ ] Commit message is clear if committed.
- [ ] Changed files reviewed.
- [ ] Unrelated changes avoided.

## Website visual check

- [ ] Desktop layout reviewed where possible.
- [ ] Mobile layout reviewed where possible.
- [ ] Important body copy remains readable.
- [ ] Section spacing remains intentional.
- [ ] CTAs remain visible.
- [ ] No obvious visual breaks.
- [ ] No placeholder text remains.
- [ ] No unwanted fake text was introduced into images.

## Copy check

- [ ] Copy is clear.
- [ ] Copy fits the brand or project.
- [ ] No generic AI language introduced.
- [ ] No generic agency clichés introduced.
- [ ] No em dashes in copy meant for RJ to publish.
- [ ] No invented facts.
- [ ] No unsupported claims.
- [ ] CTAs are clear.

## SEO check

- [ ] Page title checked where relevant.
- [ ] Meta description checked where relevant.
- [ ] H1 checked where relevant.
- [ ] H2 structure checked where relevant.
- [ ] Internal links checked where relevant.
- [ ] Alt text checked where relevant.
- [ ] No keyword stuffing.
- [ ] Search intent still supports human clarity.

## Conversion check

- [ ] Desired visitor action is clear.
- [ ] CTA path is clear.
- [ ] Trust cues are not weakened.
- [ ] Page purpose is clear.
- [ ] Service or offer clarity improved if that was the mission.
- [ ] Contact or inquiry path still works.

## Accessibility basics

- [ ] Important text is readable.
- [ ] Interactive elements are understandable.
- [ ] Images have useful alt text where appropriate.
- [ ] Color contrast was not knowingly weakened.
- [ ] Keyboard or focus behavior was not knowingly broken.

## Final report check

Before reporting back, include:

- Outcome
- What changed
- Why it matters
- Files touched or deliverables created
- Verification
- Open risks
- RJ taste calls

Do not say "done" if verification is incomplete. Say what is complete and what remains unverified.
